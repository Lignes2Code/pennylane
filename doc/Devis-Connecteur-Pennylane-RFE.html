Préambule — Contrainte de compatibilité WinDev

Le développement est recommandé en WinDev version 24 afin d'assurer la compatibilité directe avec l'application maître existante. Toute autre version (WinDev 25 à 28) n'est pas compatible sans réécriture complète des structures de données et des endpoints API.

Si la version maître est migrée en WinDev 28 ou supérieure, le développement pourra être réalisé directement dans cette version, à condition que la migration soit effective avant le début du projet.

En cas de non migration, il sera nécessaire de :
  • Recréer toutes les structures de données métier
  • Adapter l'ensemble des endpoints et appels API
